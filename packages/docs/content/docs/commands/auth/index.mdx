# ody auth

Manage stored credentials used by external integrations such as Jira and GitHub. Use this command to add credentials under named profiles or inspect the profiles currently available.

## Synopsis

```bash
ody auth <subcommand>
```

## Subcommands

| Subcommand | Description                                      |
| ---------- | ------------------------------------------------ |
| `github`   | Configure a GitHub personal access token profile |
| `jira`     | Configure a Jira email + API token profile       |
| `list`     | List configured profiles with masked credentials |

## Behavior

`ody auth` delegates to subcommands for creating and viewing credentials. Stored credentials are written to `$XDG_DATA_HOME/ody/auth.json` (typically `~/.local/share/ody/auth.json`) and saved with `0600` permissions.

Both Jira and GitHub credentials support named profiles. If no profile is specified, the default profile name is `default`.

## Examples

Configure GitHub credentials:

```bash
ody auth github
```

Configure Jira credentials for a named profile:

```bash
ody auth jira --profile work
```

List all configured credential profiles:

```bash
ody auth list
```

# ody auth github

Configure and store a GitHub personal access token for a profile. The token is prompted interactively and written to the shared auth store.

## Synopsis

```bash
ody auth github [--profile <name>]
```

## Flags

| Flag        | Alias | Type     | Default   | Description                           |
| ----------- | ----- | -------- | --------- | ------------------------------------- |
| `--profile` |       | `string` | `default` | Named profile for storing credentials |

## Behavior

1. Prompts for a GitHub personal access token.
2. Validates that the token is non-empty.
3. Saves the token under the selected profile in `$XDG_DATA_HOME/ody/auth.json`.
4. Writes the credentials file with `0600` permissions.

If the prompt is cancelled, no changes are written.

## Examples

Store credentials in the default profile:

```bash
ody auth github
```

Store credentials in a named profile:

```bash
ody auth github --profile work
```

# ody auth jira

Configure and store Jira credentials for a profile. This flow prompts for both email and API token, then saves them in the shared auth store.

## Synopsis

```bash
ody auth jira [--profile <name>]
```

## Flags

| Flag        | Alias | Type     | Default   | Description                           |
| ----------- | ----- | -------- | --------- | ------------------------------------- |
| `--profile` |       | `string` | `default` | Named profile for storing credentials |

## Behavior

1. Prompts for a Jira email address.
2. Prompts for a Jira API token.
3. Validates both values are non-empty.
4. Saves credentials under the selected profile in `$XDG_DATA_HOME/ody/auth.json`.
5. Writes the credentials file with `0600` permissions.

If either prompt is cancelled, no changes are written.

## Examples

Store credentials in the default profile:

```bash
ody auth jira
```

Store credentials in a named profile:

```bash
ody auth jira --profile work
```

# ody auth list

Display stored Jira and GitHub credential profiles with sensitive tokens masked. This helps verify what is configured and which profile is active.

## Synopsis

```bash
ody auth list
```

## Flags

This command has no flags.

## Behavior

1. Loads credentials from `$XDG_DATA_HOME/ody/auth.json`.
2. Tries to load project config to determine active Jira and GitHub profiles.
3. Prints configured Jira and GitHub profiles with masked tokens.
4. Marks active profiles and includes Jira base URL when configured.

If no credentials are configured, it prints a warning with next steps.

## Example

```text
Authentication credentials

Jira
  Profile: default (active)
  Email:   user@example.com
  Token:   ******123abc
  BaseURL: https://company.atlassian.net

GitHub
  Profile: work (active)
  Token:   ******789xyz

Done
```
