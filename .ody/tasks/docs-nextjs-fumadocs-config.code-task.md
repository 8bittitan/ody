# Task: Add Next.js and Fumadocs Configuration Files

## Description
Create all configuration files needed for the Next.js + Fumadocs documentation site to build and serve MDX content. This includes the Next.js config (with Fumadocs MDX plugin), TypeScript config, Fumadocs source collection config, PostCSS config, and Tailwind CSS config with the Fumadocs UI preset.

## Background
Fumadocs requires a specific configuration setup: the Next.js config must be wrapped with `createMDX` from `fumadocs-mdx/next`, a `source.config.ts` file defines MDX collections, and Tailwind CSS must include the Fumadocs UI preset for styling the documentation components. The TypeScript config needs a path alias for `fumadocs-mdx:collections/*` pointing to `.source/*` (generated at build time by Fumadocs).

## Technical Requirements
1. `packages/docs/next.config.mjs` -- Next.js config wrapped with `createMDX()` from `fumadocs-mdx/next`.
2. `packages/docs/tsconfig.json` -- Extends root `tsconfig.json`, adds `paths` for `fumadocs-mdx:collections/*` mapped to `./.source/*`, and includes appropriate `include`/`exclude` arrays.
3. `packages/docs/source.config.ts` -- Defines a docs collection using `defineDocs` from `fumadocs-mdx/config`, pointing to the `content/docs` directory.
4. `packages/docs/postcss.config.mjs` -- Standard PostCSS config with `@tailwindcss/postcss` plugin.
5. `packages/docs/tailwind.config.ts` -- Tailwind config using the `createPreset` from `fumadocs-ui/tailwind-plugin`, with content paths covering `app/`, `content/`, `lib/`, and Fumadocs UI node_modules.

## Dependencies
- Task `docs-workspace-package-setup` must be completed first (package.json and dependencies installed).

## Implementation Approach
1. Create `packages/docs/next.config.mjs`:
   ```js
   import { createMDX } from 'fumadocs-mdx/next';
   const withMDX = createMDX();
   /** @type {import('next').NextConfig} */
   const config = { reactStrictMode: true };
   export default withMDX(config);
   ```
2. Create `packages/docs/tsconfig.json`:
   - Extend `../../tsconfig.json`.
   - Add `compilerOptions.paths` with `fumadocs-mdx:collections/*` mapping to `./.source/*`.
   - Set `include` to `[".source/**/*", "app/**/*", "lib/**/*", "source.config.ts", "next-env.d.ts"]`.
   - Set `exclude` to `["node_modules"]`.
3. Create `packages/docs/source.config.ts`:
   - Import `defineDocs` from `fumadocs-mdx/config`.
   - Export the docs collection pointing to `./content/docs`.
4. Create `packages/docs/postcss.config.mjs`:
   - Export default config with `plugins: { '@tailwindcss/postcss': {} }`.
5. Create `packages/docs/tailwind.config.ts`:
   - Import and apply `createPreset` from `fumadocs-ui/tailwind-plugin`.
   - Set content array to cover `./app/**/*.{ts,tsx}`, `./content/**/*.mdx`, `./lib/**/*.{ts,tsx}`, and `./node_modules/fumadocs-ui/dist/**/*.js`.

## Acceptance Criteria

1. **Next.js config is valid**
   - Given `packages/docs/next.config.mjs` exists
   - When the file is imported
   - Then it exports a function (the MDX-wrapped config) without syntax errors

2. **TypeScript paths resolve**
   - Given `packages/docs/tsconfig.json` exists
   - When TypeScript resolves `fumadocs-mdx:collections/*`
   - Then it maps to `.source/*` files (generated by Fumadocs at build time)

3. **Source config defines docs collection**
   - Given `packages/docs/source.config.ts` exists
   - When Fumadocs processes it
   - Then it knows to look for MDX files in `content/docs`

4. **Tailwind includes Fumadocs preset**
   - Given `packages/docs/tailwind.config.ts` exists
   - When Tailwind processes the config
   - Then Fumadocs UI component styles are included in the output CSS

5. **PostCSS config is present**
   - Given `packages/docs/postcss.config.mjs` exists
   - When PostCSS runs
   - Then Tailwind CSS processing is enabled
